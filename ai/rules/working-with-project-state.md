# Working With Existing Project State

Read a file's current content before editing it. Don't assume a file's contents from earlier in a conversation or from documentation — documentation describes intent, the actual file is the ground truth for what's implemented.

When a design decision genuinely isn't covered by `docs/`, `CLAUDE.md`, `ai/`, or a relevant `openspec/changes/*/design.md`, ask rather than assume. Don't silently invent a convention.
